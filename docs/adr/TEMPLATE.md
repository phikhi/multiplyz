# NNNN. Titre de la décision

- **Statut** : proposed | accepted | superseded | rejected
- **Type** : arch | data | deps | security | pedago | product
- **Portée** : mineure (architect-review autonome) | majeure (sign-off humain)
- **Liens** : issue #… · PR #… · spec(s) impactée(s)

## Contexte
Le problème / la force qui pousse à décider.

## Décision
Ce qu'on décide, clairement.

## Alternatives
Options considérées + pourquoi écartées.

## Conséquences
Impacts (+ / −), specs à mettre à jour, suites.
