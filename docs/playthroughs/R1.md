## Verdict baseline / delta

**Delta vs `docs/playthroughs/R0-baseline.md`** — sur les 3 gaps que R1 (#317) ciblait explicitement
(AUDIT §5-R1, défauts A/B/C du baseline) :

- **Carte atteinte par le flux normal** : `NON` (R0) → **`OUI`** (R1). Login atterrit sur `/carte`
  (pas `/jouer`), et « Continuer » après résultats reboucle sur `/carte` (pas un nouveau niveau
  automatique). Défauts A et B du baseline sont **fermés**, confirmés par pilotage réel (canari
  vert + captures analysées), pas seulement par lecture de code.
- **Teddy visible dans la boucle de jeu** : `NON` (R0, 0 occurrence en usage normal car la carte
  n'était jamais atteinte) → **`OUI` sur la carte, toujours `NON` sur niveau/feedback/résultats**
  (R1). Le delta réel : Teddy **existait déjà** en code depuis R0.1/R1.3, mais restait invisible en
  usage normal tant que la carte n'était pas dans le flux — R1 le rend enfin **vécu**. Le reste
  (Teddy dans le jeu lui-même) est honnêtement scope R2, pas encore livré.
- **Boucle pièces vécue** (sous-partie du point économie du baseline, la partie EARN) : `NON` (pièces
  accumulées « sans but visible ») → **`OUI` pour le earn-side** : le solde monte, est vu se mettre
  à jour sur une barre persistante après chaque niveau (25→top-bar synchro instantanée, puis 10 sur
  un autre run après retour carte, delta non-tautologique prouvé par le canari). La partie
  **dépense** reste `NON` (R4, scope inchangé).
- **Art créature réel** : `NON` (R0) → **toujours `NON`** (R1 ne touchait pas ce scope ; R3).

**La valeur produit centrale de R1 est-elle atteinte par l'enfant bout-en-bout ?** (CLAUDE.md #180)
**OUI, honnêtement, pour son scope déclaré.** L'enfant vit désormais réellement : se connecter →
atterrir sur sa carte (son monde, son avatar Teddy dessus) → jouer un niveau → voir ses pièces
gagnées → revenir sur la carte et **voir son solde avoir bougé** → aller voir sa collection depuis
un vrai bouton produit. C'est très exactement la boucle « shell + nav + earn-visible » que l'épic
#317 promettait (« l'enfant doit vivre la carte et les mondes »), et le baseline R0 documentait
comme totalement absente. Les jambes **art créature réel** et **dépense des pièces** ne sont **pas**
dans le scope de R1 (elles sont R3/R4 respectivement, et R1 ne prétend jamais les livrer) — les
marquer honnêtement `NON`/deferred, plutôt que de les farder, est ce que ce gate exige (#164/#180).

- #357 (Teddy absent des écrans niveau/feedback/résultats) — déjà filé `discovered`, scope R2.

- #320/#269 (épic R4 — économie de dépense, boutique inexistante) — scope préexistant, non affecté
  par R1.
- L'artefact de rendu emoji 🪙 (disque sombre au lieu de la pièce colorée dans les captures CI) est
